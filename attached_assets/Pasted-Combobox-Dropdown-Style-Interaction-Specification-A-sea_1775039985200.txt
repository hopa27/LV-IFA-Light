Combobox Dropdown — Style & Interaction Specification

A searchable single-select dropdown built on cmdk (Command) with a connected trigger/menu design. The trigger and dropdown panel visually merge into one unified control when open, using a shared green border.

Trigger (Closed State)
Container: div — h-[44px], w-full, rounded-[8px], border border-[#BBBBBB], bg-white, px-[12px] py-[8px], flex items-center justify-between
Input: inline CommandInput — text-[16px], leading-[26px], font-['Mulish'], text-[#3d3d3d], placeholder:text-[#BBBBBB], bg-transparent, no outline/border/ring, pr-7 (clearance for chevron)
Chevron icon: absolutely positioned (right-3, top-1/2 -translate-y-1/2), w-3 h-3, text-[#006cf4] (blue), pointer-events-none, rotates 180° when open via transition-transform duration-200
State Variants
State	Trigger Style
Default	border-[#BBBBBB], bg-white
Hover (not disabled, not open, no error)	border-[#178830] (green border), chevron becomes text-[#003578] (darker blue)
Open	border-[3px] border-[#178830], bottom border removed (border-b-0), bottom corners squared (rounded-b-none), padding reduced by 2px each axis (px-[10px] py-[6px]) to compensate for thicker border
Error	border-[#d72714] (red), placeholder and text become text-[#d72714], chevron becomes text-[#d72714]
Disabled	bg-[#CCCCCC], border-[2px] border-[#ACACAC], cursor-not-allowed, opacity-100, padding adjusted (px-[11px] py-[7px]), text stays text-[#3d3d3d], placeholder becomes text-[#3d3d3d]
Dropdown Panel (Open State)
Positioning: absolute top-[100%] left-0, z-50, w-full (matches trigger width)
Border: border-[3px] border-[#178830], top border removed (border-t-0) — visually continues from trigger's green border
Corners: rounded-b-[8px] (only bottom corners rounded)
Background: bg-white, shadow-lg
Scroll area: ScrollArea with h-[200px], w-full, bg-white
Empty state: "No results found." — py-2 text-center text-sm text-[#3d3d3d]
Dropdown Items
Base: px-2 py-1.5, text-[16px], font-['Mulish'], text-[#3d3d3d], rounded-sm, cursor-default select-none
Highlighted / Hover: bg-[#003578] (dark navy), text-white
Selected (via data-[selected='true']): same as hover — bg-[#003578] text-white
Content: single <span> with flex-1 truncate
Interaction Behavior
Click trigger → opens dropdown, focuses input for type-to-filter
Type in input → filters options (via cmdk), opens dropdown if not already open
Clear input → clears the current selection, calls onChange("")
Select an item → sets value, updates input display text, closes dropdown, calls onChange(value)
Click outside (detected via containerRef and mousedown listener) → closes dropdown
Chevron → purely decorative indicator (pointer-events-none), rotates 180° when open
Wrapper
Outer container: div with relative w-full
Command wrapper: shouldFilter={false} (filtering handled manually), overflow-visible bg-transparent
